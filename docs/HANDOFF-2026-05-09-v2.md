- **I-1 — wire up GroqKeyRouter.advance() via call_with_rotation**
  (`d6acf91`)
  - `GroqKeyRouter` имел методы `current_key`, `current_key_id`,
    `advance()`, но `advance()` нигде не вызывался. На реальный
    429 от Groq бот зависал/падал.
  - Добавлен `call_with_rotation[T](router, fn)` async helper в
    `app/ai/router.py`:
    ```python
    async def call_with_rotation[T](
        router: GroqKeyRouter,
        fn: Callable[[GroqKeyRouter], Awaitable[T]],
    ) -> T:
        attempts = len(router.keys)
        last_exc = None
        for attempt in range(attempts):
            try:
                return await fn(router)
            except Exception as exc:
                if not _is_recoverable_groq_error(exc):
                    raise
                last_exc = exc
                logger.warning("groq.key_rotation", ...)
                router.advance()
        raise GroqKeysExhaustedError(...) from last_exc
    ```
  - `_is_recoverable_groq_error()` — true для `RateLimitError`,
    `InternalServerError`, и `APIStatusError` со status_code >= 500.
  - Все 6 call-site'ов (splitter, classifier, critic, courier,
    reorder, whisper) обёрнуты:
    ```python
    async def _do_call(r: GroqKeyRouter) -> SplitterResult:
        client = instructor.from_groq(AsyncGroq(api_key=r.current_key), ...)
        return await client.chat.completions.create(...)
    result = await call_with_rotation(router, _do_call)
    ```
  - 6 новых тестов в `tests/test_groq_router.py` строят
    `RateLimitError`, `InternalServerError`, `BadRequestError`
    напрямую через `httpx.Request` + `httpx.Response` (см.
    `testing-async-python` skill для шаблона). Покрывают:
    - success без advance
    - advance на 429
    - advance на 5xx
    - propagate 4xx без retry
    - exhaust pool → `GroqKeysExhaustedError` с `__cause__`
    - propagate ValueError без advance
  - Используется PEP-695 generic syntax `[T]` (Python 3.12+,
    репо pinned `>=3.12,<3.13`).
- **I-2 — mypy в CI на `app/`** (`0ae8af6`)
  - `mypy app` уже был сконфигурирован в `pyproject.toml`, но не
    был частью CI. 30 предсуществующих ошибок в 7 файлах.
  - Все 30 пофиксил:
    - `app/bot/routers/callbacks.py` — 7 мест:
      `if callback.message is not None` → `if isinstance(callback.message, Message)`.
      Aiogram-3 типизирует `callback.message: Message | InaccessibleMessage`,

      `.edit_text()` / `.edit_reply_markup()`.
    - `app/bot/services.py` — `# type: ignore[union-attr]` →
      `[attr-defined]`. mypy указывал что error code в
      `# type: ignore[union-attr]` не покрывает реально
      возникающую `attr-defined` ошибку (SQLModel column
      expressions). Plus `reminder_preset_from_offsets`
      исправлен через isinstance-narrowing optional dict values.
    - `app/shared/time.py` + `app/bot/digest.py` —
      `zi: ZoneInfo | timezone` annotation. Python's `UTC` —
      экземпляр `timezone`, не `ZoneInfo`; падало на
      `zi = UTC` после `try: zi = ZoneInfo(...)`.
    - `app/main.py` — `scheduler_handle: tuple[asyncio.Task[None],
      asyncio.Event] | None`. Раньше было `tuple[object, object]`
      из-за чего вызов `stop_inproc_scheduler(*handle)` требовал
      `# type: ignore[arg-type]`.
    - `app/ai/time_resolver.py` — удалил unused
      `# type: ignore[arg-type]` (dateparser type stubs улучшились).
    - `app/bot/digest.py` + `app/workers/scheduler.py` —
      `# type: ignore[union-attr]` → `[arg-type]` на `.order_by()`
      где SQLModel column expression передаётся в

## Известные подводные камни

### Git proxy не видит `Itosyro/plan-app`

Devin's git-manager.devin.ai/proxy не работает для этого репо
(repo private к юзеру или не зарегистрирован в Devin org config).
**Всегда** используй PAT URL для push:
```bash
git push "https://Itosyro:${GITHUB_PAT_ITOSYRO_PLANAPP}@github.com/Itosyro/plan-app.git" branch-name
```

### `git_pr` API tool отказывает на этом PAT

Сам `git_pr action="create"` возвращает «Resource not accessible
by personal access token». Причина непонятна (PAT имеет
`Pull requests: RW`). Workaround: `curl POST /repos/.../pulls`
напрямую с Bearer-токеном — это работает (проверено для PR #45).

### `git_pr action="update"` тоже отказывает на этом PAT

Та же ошибка. Используй `curl PATCH`. Чтобы избежать проблем с
многострочным телом — пиши в файл, потом подгружай через
`python3 -c json.dumps(...)` или `jq`.

### CI на этом PAT работает

`git pr_checks` возвращает результаты — для read-only операций
PAT работает нормально.

### Render free-tier деплой
- Webhook URL: `https://plan-app-t6nx.onrender.com/telegram/webhook`
- In-process scheduler (без Celery/RQ)
- DB: Neon Postgres (free-tier)
- При IDLE 15 минут — sleep, на cold start ~30 сек
- Renders auto-deploy на push в main

### `format_exc_info` warning при тестах

```
UserWarning: Remove `format_exc_info` from your processor chain
if you want pretty exceptions.
```

Это **известное** и игнорируется (см. M-4 deferred выше).
Если попытаешься удалить —
`test_e2e_partial_classify_failure_does_not_kill_batch` повиснет.

### `instructor.max_retries=2` — артефакт

В каждом Groq call site используется `max_retries=2`. Это
instructor's валидация-retry, не HTTP-retry. Не трогай без
понимания. Affects M-8 (1.54s после 400-fix; больше не сократить
без удаления `max_retries`).
