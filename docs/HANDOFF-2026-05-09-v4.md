## Что нашёл (главное — почему раньше не заметили)

Подавляющее большинство находок прошло мимо предыдущих ревью
потому, что **тестовый набор не покрывает соответствующие
сценарии**:

* SQLite не принуждает FK по умолчанию → тесты на `delete_task`
  проходят, прод-Postgres сломается.
* Тесты `cb_settings_set` проверяют только структуру клавиатуры,
  не симулируют клик.
* Нет concurrent-webhook теста.
* Нет теста на `сегодня в HH:MM` после `now`.
* Нет теста на повторный `/start` после онбординга.

### Критические (silently broken in production)

1. **R-NEW-C-1** — `/settings` UI не может менять время дайджестов
   (`settings.py:230-240`: `split(":")` ломается на `08:00`).
2. **R-NEW-C-2** — `сегодня в 10:00` после 14:00 → задача на
   следующую неделю (`time_resolver.py`: +7 дней даже для "сегодня").
3. **R-NEW-C-3** — `delete_task` сломается в Postgres из-за
   `task_events.task_id` без `ON DELETE` (тесты SQLite пропускают).
4. **R-NEW-C-4** — `_pipeline.py:151` передаёт `[]` в `classify_intent`
   как `user_categories` → классификатор плодит дубли категорий.
5. **R-NEW-C-5** — Webhook idempotency TOCTOU: два параллельных
   запроса с одним `update_id` оба проходят check, второй падает с
   `IntegrityError` → 500 → Telegram retry'ит вечно.
6. **R-NEW-C-6** — Voice handler не грузит
   `default_reminder_offsets`; голосовые юзеры всегда получают
   глобальный default preset.

### Important (8) и Minor (9) — см. `docs/REVIEW-2026-05-09-v2.md`.

## Предложенный порядок фиксов (cheapest, highest impact first)

1. **C-5** webhook idempotency race (Telegram может бомбить
   удалённо; самый дешёвый фикс).
2. **C-1** settings UI (одна строка + тест).
3. **C-4** classifier user_categories (компаундится по времени).
4. **C-6** voice reminder offsets (одна копия блока).
5. **C-2** time-resolver "сегодня в HH:MM" (фрагмент-чек + тест).
6. **C-3 + I-7** FK / ON DELETE миграция (одна alembic-ревизия,
   двухстадийная).
7. **I-1** callback `int()` hardening (механический фикс).
8. **I-2** category/horizon race.
9. **I-3** re-onboarding.
10. **I-4** digest tick robustness.
11. **I-5** reminder per-row commit.
12. **I-6** `/today` pagination.
13. **I-8** backpressure.
14. Минорные (M-1..M-9) одним PR housekeeping.

Каждый шаг тестируем независимо. Phase-5 (Mini App) от этого не
зависит.

## Конвенции команды (что НЕЛЬЗЯ ломать)

* Docstrings — на английском, комменты сложной логики — по-русски
  ("почему", не "что").
* **No `Any` / `getattr` / `setattr`** для обхода типов. Если
  кажется, что они нужны — значит ты не понял типы.
* **No `print()`** в проде.
* **Импорты — сверху файла**, не внутри функций.
* **AI-промпты — только в `app/ai/prompts/<name>.md`**, не inline
  в коде.
* **`parse_mode="Markdown"`** запрещён для любого user-controlled
  текста (см. C-2 v1, R-NEW-C-1 v2).
* **Naive UTC** для всех DB-колонок. См. `app/shared/time.py` —
  `utcnow_naive()` / `to_naive_utc()` / `format_due_local()`.
* **Idempotency** на каждом webhook handler.
* **Allow-list** на каждой границе (settings, callback data,
  LLM output).
* Миграции **двухстадийные** при удалении колонок / смене
  ON DELETE.
* PR pre-commit hooks: `pre-commit install` после `uv sync`.

## Известные процессные ограничения

* `git_pr(action="create")` падает с "Resource not accessible by
  personal access token". Используй `curl POST` через `${GITHUB}`
  токен. Все остальные `git_pr` действия работают.
* Тесты используют SQLite in-memory (`aiosqlite`). FK constraint
  enforcement выключен по умолчанию. Если хочешь поймать FK-баги
  до прода — включи `PRAGMA foreign_keys=ON` в conftest или
  переключи на in-memory Postgres (`testcontainers`). Это отдельная
  работа.
* `gh` CLI **не работает** для GHES proxy. Только git + curl + token.
