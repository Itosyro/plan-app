## Что закрыто в этой сессии

### Critical

* **C-1 (PR #53)** — `/settings` → выбор времени дайджеста (08:00,
  21:00 и т.д.) больше не падает с «Неверный формат».
  `parse_set_callback` теперь использует `split(":", 3)`, поэтому
  `morning_digest_at:08:00` парсится правильно. Тесты проходят
  по каждой кнопке клавиатуры.
* **C-2 (PR #56)** — «сегодня в 10:00» в 14:00 теперь резолвится в
  *сегодня*, а не через неделю. Перед `+7d` rollover проверяем
  явный маркер «сегодня/сейчас/today» в исходном тексте.
  «во вторник» во вторник по-прежнему уходит на следующий вторник.
* **C-3 + I-7 (PR #57)** — `delete_task` больше не FK-violate'ит на
  Postgres. Migration `0007` ставит ON DELETE policies на ВСЕ
  FK-констрейнты:
  * `task_events.task_id`, `reminders.task_id`, `*.user_id` →
    `CASCADE`.
  * `tasks.{category_id, horizon_id, source_inbox_id}`,
    `notes.{category_id, source_inbox_id}`, `ai_runs.inbox_id`,
    `telegram_updates.user_id` → `SET NULL`.
  * Модели обновлены для `task_events.task_id` и
    `reminders.task_id` так, что `create_all` в тестах тоже
    производит CASCADE — позволяет писать тесты с
    `PRAGMA foreign_keys = ON`.
* **C-4 (PR #54)** — `_pipeline.run_pipeline` теперь подгружает
  существующие категории пользователя через `get_user_categories`
  и передаёт их в классификатор. Дубли вида «Работа / работа /
  Рабочее» больше не плодятся.
* **C-5 (PR #52)** — webhook idempotency больше не TOCTOU.
  `mark_update_processed` теперь делает атомарный INSERT и ловит
  `IntegrityError` — параллельные запросы с одним `update_id`
  получают консистентный результат, никаких 500-х.
* **C-6 (PR #55)** — voice handler грузит
  `default_reminder_offsets` из `UserSettings` ровно как
  text.py. Голосовые юзеры теперь получают свои reminder presets,
  а не глобальный default.

## Что ещё точно нужно сделать

Приоритеты по убыванию (subjective):

1. **Смержить PR'ы 50-57** в этом порядке. После каждого мержа
   запустить `uv run pytest -q` в main — должно быть 207 green.
   PR #50 (docs review) и #51 (handoff v4) можно мержить в любой
   момент — изменений кода нет.
2. **Important I-1 .. I-6, I-8** из v2 review. Самые важные:
   * **I-1** — callback handlers без try/except на `int()`
     (старые, до C-1, в `_settings.py`/`_today.py`/etc).
   * **I-3** — повторный `/start` крашит, потому что
     `get_or_create_user` ожидает первый раз.
   * **I-4** — digest scheduler tick drift: если scheduler
     просыпается каждые 60s, но пользователь обновил
     `morning_digest_at` посреди интервала, дайджест может
     пропуститься на этот день.
   * **I-5** — reminder worker не идемпотентен на crash:
     перезапуск может выслать одно и то же напоминание дважды.
   * **I-6** — `/today` шлёт N задач = N сообщений. Нужно
     слить в одно сообщение ≤ 4096 символов (см. UX правило
     в `defensive-programming/SKILL.md`).
   * **I-8** — нет backpressure на webhook: при пиковой
     нагрузке Telegram retries будут наслаиваться.
3. **Minor M-1 .. M-9** из v2 review. Hygiene-уровень.
4. **Phase 5 (mini-app)** — никто ещё не трогал. План в
   `docs/PLAN.md` и `plan-python-detailed.md`.
