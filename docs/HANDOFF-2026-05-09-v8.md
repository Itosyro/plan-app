## 1. Что мы создаём — продуктовое видение

### 1.1. История пользователя

Юзер (это владелец проекта, agace2) — пишет в боте `@daylirobot`
голосом или текстом. Хаотично. «утром пробежка 5 км, потом
совещание в 11, на этой неделе сдать отчёт, обед через час
напомни, идея — можно сделать функцию X в боте, во вторник врачу».

Бот:
1. Транскрибирует через Whisper (если голос).
2. Splitter режет на 5 атомарных интентов.
3. Time Resolver (детерминистический Python) превращает «утром»,
   «через час», «во вторник» в конкретные `datetime` с учётом
   часового пояса юзера и его персональных «morning_anchor» /
   «evening_anchor».
4. Classifier (умная LLM) каждому интенту ставит:
   `category` (Здоровье / Работа / Идея / Здоровье...),
   `horizon` (today/tomorrow/week/month/year/someday),
   `priority` (low/medium/high), `is_task` (или это note?),
   `reminder_offsets` (если просили напомнить).
5. Critic (другая LLM) перепроверяет classifier'а по уверенности
   (порог 0.7 по умолчанию, тумблер в /settings).
6. Сохраняет в Postgres: задачи в `tasks`, заметки в `notes`,
   напоминания в `reminders`.
7. Courier отвечает в Telegram: подтверждение («Понял, мой
   господин») + резюме сделанного («3 задачи + 1 напоминание +
   1 идея сохранены»).
8. In-process scheduler каждую минуту проверяет `reminders` и
   шлёт уведомления когда `fire_at <= now()`.
9. В нужное время утром (08:00 локально) и вечером (21:00)
   шлёт дайджест: «вот твои задачи сегодня».

Команды: `/today /tomorrow /week /month /year /someday /notes
/categories /settings /help /start`.

Inline-кнопки на задачах: «Готово», «Перенести», «Удалить»,
«Сменить категорию».

### 1.2. Главные ценности

- **Низкий порог входа.** Юзер не учит синтаксис — просто говорит.
- **Русский первого класса.** Whisper + dateparser + Russian few-shot
  промпты.
- **Не теряет данные.** Webhook idempotency (TelegramUpdate),
  reminder claim-pattern, ON DELETE CASCADE для FK.
- **Backpressure.** Per-user semaphore = 1, global = 8 — спам не
  кладёт бота.
- **Понятен пользователю.** Дайджест в его таймзоне, дефолты
  настраиваются в /settings, всё на русском.

| `git commit --amend` на чужих коммитах | Перепишет историю |
| Менять тесты чтобы зелёнить | Если уверен что тест неверный — обнови, объяснив; иначе чини код |
| Коммитить `.env` / secrets в plain text | Ну ты понял |
| Использовать `gh` CLI | Используй `git_pr` + `git` tools или curl с GitHub API |
| Pre-commit hooks | Не настроены, не настраивай (тормозит агентов) |
| Менять `getattr`-стиль в callbacks без `parse_task_callback` | Уже починили в I-1, не сломай |
| Шумные комментарии в стиле «added to handle X» | Запрещено core-инструкциями. Комментируй *код*, не *диф* |

| Коммитить план / TODO в репо | План в `docs/PLAN.md`, ad-hoc TODO — в session `todo_write` |

### 5.2. Известные грабли проекта

1. **`asyncio.create_task` без backpressure** — мы починили
   в I-8 для `run_pipeline`, но если будешь делать новые
   фоновые задачи — оборачивай аналогично.
2. **`session.execute(update(...))` возвращает Union с
   `Result[Any]`** — `.rowcount` есть только на `CursorResult`.
   Решай через `assert isinstance(..., CursorResult)`. Не
   `getattr`.
3. **SAVEPOINT не reliable под `asyncio.gather`** — для race-safe
   upsert используй `INSERT ... ON CONFLICT DO NOTHING`
   (см. `app/bot/services/tasks.py::get_or_create_category`).
4. **`Insert` mypy не expose'ит dialect-методы** — union
   `_PgInsert | _SqliteInsert`.
5. **Render Free засыпает через 15 мин** — внешний пинг
   `cron-job.org → /healthz` каждые 5 мин держит тёплым.
   Если test'ишь locally — учти что в проде latency на cold
   start ~30 сек.
6. **In-memory SQLite в тестах не enforces FKs** — `PRAGMA
   foreign_keys = ON` в `conftest.py`.
7. **`UserSettings.user_id` — PK, не auto-increment**: при
   re-`/start` нужен SELECT-then-INSERT, не плейн `add()`.
8. **Day-1 digest safeguard:** если юзер онбордится после
   `morning_digest_at` — первый дайджест придёт ЗАВТРА.
9. **`pymorphy3` / `razdel` УДАЛЕНЫ** в M-1 — `dateparser`
   достаточно. Не возвращай их обратно без причины.
10. **Stuck `processing`-rows в `reminders`** — оставляем для
    ручного triage. Не делай auto-cleanup без обсуждения.


## 9. Тестирование — что мокаем, что нет

| Что | Как тестируем |
|---|---|
| Groq LLM-вызовы | `respx.mock()` мокает `https://api.groq.com/...` |
| Whisper | то же — `respx` |
| Telegram Bot API | `aiogram.types.Update.model_validate(...)` + проверяем что хендлер не падает + что в БД появилась запись |
| Postgres | aiosqlite in-memory + `init_db()` в фикстуре |
| In-process scheduler | вызываем `tick_reminders(bot, now=fixed_now)` напрямую |
| Time | передаём `now=...` в функции, не используем `datetime.utcnow()` |
| Asyncio semaphore (I-8) | monkey-patch `_run_pipeline_inner` |

Скилл `testing-async-python/SKILL.md` объясняет паттерны.

## 13. Если что-то пойдёт не так

### 13.1. CI красный после твоего PR

1. Прочитай логи через `git ci_job_logs job_id=<from pr_checks>`.
2. Если простая ошибка — исправь, push, ещё раз CI.
3. Если 3+ попытки — сообщи юзеру через `message_user(block=true)`,
   но не останавливайся, иди дальше делать другие PR'ы.
4. Никогда не мерж'и сломанный CI.

### 13.2. Возникла мерж-конфликт

1. `git pull --rebase origin main` на твоей ветке.
2. Если конфликты тривиальные (импорты, lockfiles) — резолви сам.
3. Если semantic — пиши юзеру, но опять же продолжай работу
   на других PR'ах.

### 13.3. Тест падает локально

1. Активируй `systematic-debugging/SKILL.md`.
2. Найди корневую причину, не «накладывай пластырь».
3. Если уверен что тест неверный — обнови и объясни в commit.

### 13.4. Закончился контекст

1. **Не паникуй.** Создай `docs/HANDOFF-2026-05-09-v9.md` со
   всем тем же уровнем детализации что и этот файл.
2. Push в финальный docs PR.
3. `message_user(block=true)` с резюме сделанного и ссылкой
   на handoff.

### 13.5. Render auto-deploy упал после мержа

1. Юзер получит уведомление от Render.
2. Ты узнаешь по `/healthz` 5xx.
3. Срочно: revert commit или fix-forward, новый PR.
4. **Сообщи юзеру** через `message_user(block=true)`.
