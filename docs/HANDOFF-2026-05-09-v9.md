## 1. Что было сделано в этой сессии

### Phase 5 — Mini-App + streaming (PR #64, #65)

- **Backend:** `/api/{me,horizons,categories,tasks,notes,inbox}`
  с HMAC-валидацией `X-Telegram-Init-Data` (TTL 24 ч + grace).
  Переиспользует `app/bot/services/*` для аудит-trail.
- **Frontend:** `webapp/` — React 18 + Vite 5 + Tailwind 3 + TS
  strict, mobile-first Todoist-style UI (pill-табы горизонтов,
  фильтр категорий, карточки с checkbox / приоритет / срок /
  Перенести / Удалить). Тема наследуется от Telegram.
- **Streaming-replies:** `app/bot/streaming.py` —
  `editMessageText` построчно с `sendChatAction("typing")` +
  rate-limit-aware retry.
- **Hotfix #65:** Render service создавался руками, `render.yaml`
  игнорился. Через Render API переписали buildCommand чтобы он
  собирал `webapp/dist/`.

### Phase 6 — Bot API 10.0 polish

- **6.1 Reactions** (`app/bot/reactions.py`, PR #66): 👀 на
  получение → 🎉 на успех → 😢 на ошибку. Allow-list эмодзи,
  best-effort (Telegram-ошибки не валят пайплайн).
- **6.2 Quote replies** (`app/bot/quote_replies.py`, PR #67):
  ответ бота прикрепляется к user message с tap-to-jump
  стрелкой через `reply_parameters.quote` (Bot API 7.0+).
- **6.3 Pinned morning digest** (`app/bot/pinned_today.py`,
  PR #69): утренний дайджест пинится, в течение дня live-update
  через `editMessageText` при mark-done (callback ИЛИ Mini-App
  PATCH). Migration 0008 добавляет `pinned_morning_*` на
  `user_settings`.
- **6.4 CloudStorage** (`webapp/src/lib/storage.ts`, PR #68):
  Mini-App UI prefs (`last_horizon`, `last_category`) персистятся
  через `WebApp.CloudStorage` с откатом на `localStorage`.

### Прод-операции

- **Migrations 0002-0008** накатаны на Neon (раньше прод был на
  0001, схема существовала только частично — categories /
  horizons / tasks / task_events / notes / reminders / ai_runs
  отсутствовали).
- **`startCommand` обновлён** до
  `uv run alembic upgrade head && uv run uvicorn ...` —
  изменение и в `render.yaml`, и в живом Render service через

## 3. Что делать дальше (Phase 5.4+)

Приоритет: **самое user-visible для углубления Mini-App.**

### 5.4 Counts endpoint

`GET /api/tasks/counts` → `{today: 3, tomorrow: 1, week: 8,
month: 15, sometime: 5}`. Один SQL-запрос с `CASE WHEN due_at
BETWEEN ... THEN ... END` или 5 параллельных count-запросов.
Frontend показывает счётчики на pill-табах: «Сегодня (3)».

### 5.4 Drag-n-drop reorder между горизонтами

`@dnd-kit/core` + `@dnd-kit/sortable`. Колонки = горизонты,
drag меняет horizon через `PATCH /api/tasks/:id`. Optimistic
update + rollback на ошибку.

### 5.5 Calendar / Kanban view

FullCalendar/react ИЛИ простая kanban-доска. События по
`due_at`, drag по сетке двигает `due_at` через PATCH.

### 5.6 Карточка задачи

Модалка с TaskEvent-историей и оригиналом inbox_entry (текст /
voice player). Полный CRUD: title, due_at, priority, category.

## 6. Анти-паттерны (грабли, на которые наступали)

1. **Render service создан руками → `render.yaml` игнорится.**
   Любое изменение buildCommand/startCommand нужно делать через
   Render API. Скрипт:
   ```bash
   curl -X PATCH -H "Authorization: Bearer $RENDER_API" \
     -H "Content-Type: application/json" \
     -d '{"serviceDetails":{"envSpecificDetails":{"startCommand":"..."}}}' \
     https://api.render.com/v1/services/srv-d7uohcf7f7vs73crmk3g
   ```

2. **Миграции на прод не накатывались.** До этой сессии
   `startCommand` = только uvicorn, без alembic. Любой PR с
   миграцией работал в локали и в CI, но в проде создавал
   рассинхрон между моделью и схемой. Сейчас исправлено
   (`alembic upgrade head &&` в startCommand).

3. **`_FakeBot.send_message()` в тестах должен возвращать
   объект с `.message_id`,** иначе `pinned_today.py` не
   запишет state. См. `tests/test_digest.py` и
   `tests/test_e2e_phase4.py` — оба используют
   `SimpleNamespace(message_id=…)`.

4. **DATABASE_URL — это приватный neon URL,** есть только в
   Render env-vars. Из Render API: `GET /services/{id}/env-vars`,
   достаём `DATABASE_URL`, экспортируем локально, прогоняем
   `alembic`. **Никогда не коммитим в git.**
