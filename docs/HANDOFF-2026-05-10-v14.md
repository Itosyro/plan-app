**Открытые места** PR-C, которые могут стать техдолгом:
- В `NoteDetail` `category_id: null` через BottomSheetSelect не «чистится»
  для существующей заметки — backend `PATCH` пропускает `None` (не отличает
  «не передавали» от «очистить»). Если юзер пожалуется — добавить
  семантический маркер (например, sentinel `-1`) или отдельный
  `DELETE /api/notes/{id}/category`. Сейчас просто `setDraftCategoryId(null)`
  локально, без сетевого запроса. UI-уровень: при тапе «Без категории»
  на существующей заметке ничего не происходит. Не критично, но запиши.
- В `NotesList` нет server-side фильтра по категории. Я добавил клиентский
  substring-search; если списков станут тысячи — переделать на серверный поиск
  (`GET /api/notes?q=…`) и пагинацию.
- `apiClient.createNote` вызывается ТОЛЬКО при blur заголовка (если title
  непустой). Если юзер сразу нажмёт «Назад» из режима create — заметка
  не создаётся. Это by design (нет смысла создавать пустые), но юзер может
  ожидать «autosave on type». Пока не делал — спросить юзера.

## 3 · Roadmap → что осталось

Все детали PR-D / PR-E / PR-F в **v13 §4-§5** актуальны. Ниже — обновления и
уточнения, появившиеся за PR-B/PR-C.

### PR-D — Корзина (24h soft-delete)

**Что осталось без изменений** (v13 §4 PR-D):
- Новое поле `deleted_at: datetime | None` на `Task`, `Note`, `Reminder`.
- Alembic-миграция `0009_soft_delete.py` с partial-index'ом
  `WHERE deleted_at IS NULL` для горячих SELECT-путей.
- Все `DELETE`-ручки в `app/api/routers/{tasks,notes,reminders}.py` —
  `obj.deleted_at = utcnow_naive()` вместо `session.delete(obj)`.
- Все 6 SELECT-путей `app/bot/services/*.py` + `app/api/routers/*.py` —
  добавить `.where(Task.deleted_at.is_(None))` (и аналогично).
- Worker `app/jobs/purge_trash.py` с `app/scheduler/jobs/purge_trash_job.py` —
  каждый час пробегает строки старше 24h и физически удаляет.
- `app/api/routers/trash.py` (новый) — `GET /api/trash`, `POST /api/trash/{kind}/{id}/restore`.
- `webapp/src/components/TrashPage.tsx` — список + restore-кнопка.

**Обновления по PR-C (это важно для PR-D):**
- `DELETE /api/notes/{id}` уже есть (`app/api/routers/notes.py:48-66`).
  В PR-D его надо переписать на soft-delete. Тест `test_delete_note` будет
  обновлён — на 204, но проверяем что запись осталась с `deleted_at != None`.
- `NoteDetail` уже умеет deleteNote — UI флоу не меняется. Только надо
  убедиться что список не показывает soft-deleted строки (но это в API-уровне).
- Frontend `apiClient.deleteNote` уже корректен.

**Дополнительные acceptance:**
- Миграция тестируется на свежей чистой БД (alembic upgrade head + downgrade)
  и на seed-данных.
- Worker идемпотентен: повторный прогон не падает.
- GC не трогает строки с `deleted_at = NULL`.
- В UI Trash — отдельные секции «Задачи / Заметки / Напоминания», каждая
  с timestamp `<X часов назад> · удалено`.

### PR-E — Бот: дружелюбные тексты + чек-карточка с inline-keyboard

## 5 · Операционка (то же, что v13 §6, но обновлено)

### 5.1 Git push + PR + merge через curl

```bash
# Push (если git-manager-proxy 403'ит)
git push "https://Itosyro:${GUTHUB}@github.com/Itosyro/plan-app.git" HEAD:<branch>

# Create PR
cat > /tmp/pr-body.json << 'EOF'
{"title":"...","head":"<branch>","base":"main","body":"..."}
EOF
curl -s -X POST -H "Authorization: token ${GUTHUB}" \
  -H "Accept: application/vnd.github+json" \
  https://api.github.com/repos/Itosyro/plan-app/pulls \
  -d @/tmp/pr-body.json

# Wait CI (use builtin tool)
# git(action="pr_checks", repo="Itosyro/plan-app", pull_number=N, wait_mode="all")

# Merge (squash)
curl -s -X PUT -H "Authorization: token ${GUTHUB}" \
  https://api.github.com/repos/Itosyro/plan-app/pulls/<N>/merge \
  -d '{"merge_method":"squash"}'
```

### 5.2 Preview HTML для скриншотов

`webapp/dist/preview.html` — мокает `window.Telegram.WebApp` + fetch-mock с
fixtures (tasks + categories + notes). Подаётся через `python3 -m http.server 9000`.
