## 1 · Что сделано в PR-E

См. `docs/PROGRESS.md` запись от 2026-05-11. Ключевые изменения:

### 1.1 — `courier_respond` теперь отдаёт `(text, keyboard)`

```python
# app/ai/courier.py
async def courier_respond(
    router, items: list[SummaryItem], *, mode="mix", style="neutral"
) -> tuple[str, InlineKeyboardMarkup | None]:
    ...
```

`SummaryItem(kind, title, category_name, persisted_id, status)` — это
одна строка карточки распознанного. `build_summary_keyboard(items)`
рендерит inline-клавиатуру одной кнопкой-строкой на item, с
callback_data `summary:toggle:<kind>:<id>`. Префиксы:
`TASK_PENDING_PREFIX = "☐ "`, `TASK_DONE_PREFIX = "✅ "`,
`NOTE_PREFIX = "📄 "`.

### 1.2 — `run_pipeline` возвращает tuple

```python
# app/bot/routers/_pipeline.py
PipelineReply = tuple[str, InlineKeyboardMarkup | None]

async def run_pipeline(...) -> PipelineReply: ...
```

Все ранние return (reorder, empty units, survivors) тоже возвращают
tuple. Callers в `text.py`/`voice.py` анпакуют и передают
`reply_markup=keyboard` в `stream_reply(..., reply_markup=keyboard)`.

`stream_reply` крепит markup *только* на финальном edit — иначе
клавиатура мигала бы в середине streaming-эффекта.

### 1.3 — Callback `summary:toggle:<kind>:<id>`

`app/bot/routers/callbacks.py::cb_summary_toggle`:
- kind=task → `mark_task_done` / `mark_task_undone`, флипает префикс
  ☐↔✅ в клавиатуре, рефрешит pinned-morning digest. Event `reopened`
  пишется в `task_events`.
- kind=note → toast «Заметка — не требует действий», БД не трогается.

Парсинг payload — `parse_summary_toggle_callback(data)`. Никаких
unguarded `int(parts[N])` в хендлерах (R-NEW-I-1 дисциплина).

### 1.4 — `concretize_tasks` (опт-ин)

- `app/db/models.py::UserSettings.concretize_tasks: bool = False`
- Миграция `alembic/versions/2026_05_11_1100-0010_concretize_tasks.py`
  (SQLite-aware: `"0"` для sqlite, `false()` для остальных).
- Бот: команда `/settings` → новый ряд «Первый шаг в задачах» (on/off).
- Mini-app: `SettingsPage.tsx` → row «Первый шаг» с иконкой `ListChecks`.
- API: `UserSettingsOut.concretize_tasks: bool` +
  `UserSettingsUpdateIn.concretize_tasks: bool | None`. `patch_me`
  конвертит bool↔«on»/«off» на границе (бот хранит как bool, mini-app
  шлёт натуральный bool).

### 1.5 — `first_step` в классификаторе

- `ClassifierResult.first_step: str | None`.
- Промпт `app/ai/prompts/classifier.md` обновлён: новый раздел
  «First step (optional, only for tasks)» с правилами и примерами.
- `persist_classification(..., concretize_tasks: bool)` склеивает
  `first_step` в `Task.description` префиксом `Шаг 1: …` если
  toggle включён.

### 1.6 — Шаблоны подтверждений

`TEMPLATES` в `courier.py` переписаны:
- ≤1 эмодзи на фразу (тест `test_templates_no_emoji_parade`).

## 6 · Что строго НЕЛЬЗЯ

- `git push --force` на main или master.
- `git commit --no-verify` — pre-commit хуки настроены не зря.
- `git commit --amend` после push.
- `git add .` — добавляет лишнее.
- `getattr` / `setattr` для доступа к атрибутам моделей — это сигнал
  что ты не понял типы. Используй явные if-маппинги
  (см. `app/bot/routers/settings.py::_setting_value`).
- Менять тесты только для того чтобы они проходили. Если тест
  проверяет старый API surface который PR явно меняет — можно
  обновить, *с описанием* в комменте теста почему.
- Динамические Tailwind классы вроде `bg-${tone}-500` — JIT не
  подхватит. Используй safelist в `tailwind.config.cjs` либо
  conditional полные классы.
