## 4 · Технический долг

- **`session.execute()` DeprecationWarning**: ~10 мест в `tasks.py` и `inbox.py` используют `session.execute()` вместо `session.exec()`. Не критично, работает, но pytest кричит. Отдельный PR-L.
- **`_try_reorder` legacy**: `app/ai/reorder.py` + `detect_reorder()` всё ещё вызывается в pipeline как fallback *после* intent router'а. Если intent router стабилен — можно убрать `_try_reorder` и оставить только `detect_intent` + `_execute_reorder_horizon`. Тогда удалить `reorder.py`, `prompts/reorder.md`.
- **Callback state для I2 intents**: при multi-match disambiguation, callback data содержит только `intent:task_id`. Для I2 intents (rename, set_due, etc.) дополнительные данные (new_title, new_due_raw) теряются. Решение: persisted intent state (Redis/in-memory dict с TTL) — PR-I3.

## 5 · PR-I3 — следующий PR (детальный спек)

### 5.1 LAST_TASK анафоры

In-memory dict `LAST_TASK: dict[int, tuple[int, float]]` — `{user_id: (task_id, timestamp)}`.
TTL 60 секунд. Обновляется при:
- `persist_classification` (новая задача).
- `execute_edit` (любой executor).
- Callback `cb_edit_resolve`.

Если `intent.task_query` пустой → проверяем `LAST_TASK[user_id]`. Если найден и TTL не истёк → используем.
Иначе → «Не понял, какую задачу ты имеешь в виду.»

### 5.2 Multi-intent

Юзер может сказать «сделал пробежку и перенеси отчёт на завтра».

Поток:
1. `split_message(text)` → `["сделал пробежку", "перенеси отчёт на завтра"]`
2. Для каждого unit: `detect_intent(unit)`.
3. Выполнить каждый intent отдельно.
4. Собрать все reply в один текст.

### 5.3 Undo

Миграция: `task_edit_snapshots` таблица:
```sql
id SERIAL PRIMARY KEY,
task_id INT REFERENCES tasks(id),
user_id INT REFERENCES users(id),
field TEXT,          -- 'title', 'priority', 'due_at', 'status', 'horizon_id', 'category_id'
old_value TEXT,
new_value TEXT,
created_at TIMESTAMP DEFAULT now()
```

После каждого executor → сохранить snapshot → ответить с inline-кнопкой `[Отменить]`.
Callback `edit:undo:<snapshot_id>` → восстановить old_value.
TTL: snapshot удаляется через 5 минут (scheduler или lazy check).

### 5.4 list_completed_today

Новый read-only intent. Возвращает список задач с `status=done` и `TaskEvent.kind=completed` today.

### 5.5 Multi-match state persistence

Для I2 intents при >1 match: сохранить `EditIntent` в `PENDING_EDITS: dict[int, EditIntent]` (по user_id).
Callback `cb_edit_resolve` → достать intent из `PENDING_EDITS`, заменить task_id, выполнить.
TTL: 60 секунд.

### 5.6 Разделение PR-I3 на подзадачи

- [ ] LAST_TASK dict + integration в `execute_edit`.
- [ ] Multi-intent через `split_message` loop.
- [ ] `list_completed_today` executor.
- [ ] PENDING_EDITS для multi-match I2 state.
- [ ] Миграция `task_edit_snapshots` + Undo flow.
- [ ] Тесты ~10 кейсов.
- [ ] ETA: 3-4 часа.

## 14 · Лессоны из этой сессии

- **Session isolation для executors.** Передавать Task objects между сессиями вызывает `InvalidRequestError: Object already attached to session`. Решение: executors принимают `task_id` + `user_id`, каждый создаёт свою `session_scope()` и lookup'ит задачу.
- **E2E тесты ломаются при добавлении LLM-шага.** Когда pipeline добавляет `detect_intent()` перед `_try_reorder`, все E2E mock-sequences сдвигаются на 1. Создай `_intent_response()` helper и вставляй его первым в каждый `_CallTracker`.
- **`git_pr(action="create")` возвращает 403** — fallback через curl API к GitHub. Токен `${GUTHUB}` имеет полный доступ.
- **`git push origin` через proxy даёт 500/403** — fallback: `git push https://devin:${GUTHUB}@github.com/Itosyro/plan-app.git`.
- **Ruff auto-fix надёжен.** `ruff check --fix` для import sorting и unused import removal работает безопасно. После fix — всегда проверь `ruff check` без `--fix`.
- **dateparser.parse с русским.** `languages=["ru"]` + `settings={"PREFER_DATES_FROM": "future"}` корректно парсит «завтра в 10», «в пятницу», «через час». Результат — aware datetime, нужно `to_naive_utc()` перед сохранением.
- **Не забывай удалять неиспользуемые imports** после рефакторинга (sqlmodel select, AsyncSession и т.д.). Ruff поймает, но лучше сразу.

## 16 · Соглашения о коде
