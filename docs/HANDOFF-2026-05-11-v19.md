## 2 · Детали PR-I4

### 2.1 TaskEditSnapshot модель
`app/db/models.py` — новый класс `TaskEditSnapshot(SQLModel, table=True)`:
- `id: int` (PK, auto)
- `task_id: int` (FK → tasks.id, ON DELETE CASCADE, indexed)
- `user_id: int` (FK → users.id, indexed)
- `field: str` (max_length=32) — имя поля (status, title, priority, due_at, horizon_id, category_id, deleted_at)
- `old_value: str | None` — значение до изменения
- `new_value: str | None` — значение после изменения
- `created_at: datetime` — timestamp для lazy TTL

### 2.2 Alembic migration 0011
`alembic/versions/2026_05_11_1700-0011_task_edit_snapshots.py` — CREATE TABLE.
Ревизия `0011`, down_revision `0010`. Написана вручную (autogenerate не работает с SQLite).

### 2.3 Executor return type change
Все 9 executor'ов изменили сигнатуру:
- Было: `async def _execute_*(task_id, user_id, ...) -> str`
- Стало: `async def _execute_*(task_id, user_id, ...) -> tuple[str, int | None]`

Каждый executor вызывает `_save_snapshot()` после мутации и возвращает `(reply_text, snapshot_id)`.
Error/no-change cases возвращают `(reply_text, None)`.

`_dispatch_single()` тоже возвращает `tuple[str, int | None]`.

### 2.4 Snapshot helpers
```python
_UNDO_TTL_SECONDS = 300  # 5 minutes

async def _save_snapshot(task_id, user_id, field, old_value, new_value) -> int | None
def _undo_keyboard(snapshot_id: int) -> InlineKeyboardMarkup
```

### 2.5 Inline кнопка [Отменить]
В `execute_edit()` — если `snap_id is not None`, строит `_undo_keyboard(snap_id)` и возвращает как reply_markup.
Аналогично в `cb_edit_resolve()` (disambiguation callback).

Callback data format: `edit:undo:<snapshot_id>`.

### 2.6 Undo handler
`cb_edit_undo` в `callbacks.py`:
1. `parse_edit_undo_callback(data)` — извлечение snapshot_id (вне `create_router()` — проходит regression test `test_callback_handlers_no_unguarded_int_parts`).
2. Load snapshot by id + user_id.
3. Lazy TTL: `(utcnow_naive() - snap.created_at).total_seconds() > 300` → «Время для отмены истекло (5 мин).», кнопка удаляется.
4. Load task by snap.task_id.
5. `_apply_undo(task, snap)` — восстанавливает old_value в зависимости от field:
   - `status` → task.status = old_value
   - `deleted_at` → task.deleted_at = None
   - `title` → task.title = old_value
   - `priority` → task.priority = old_value
   - `due_at` → task.due_at = datetime.fromisoformat(old_value)
   - `horizon_id` → task.horizon_id = int(old_value)
   - `category_id` → task.category_id = int(old_value)
6. Delete snapshot row.
7. Reply confirmation, remove inline keyboard.

## 4 · Архитектура intent detection (итог I1–I4)

```
Текст → detect_intent(full text)
         ├─ edit intent (10 шт) → execute_edit()
         │   ├─ list_done → _execute_list_completed_today (read-only, no undo)
         │   ├─ query empty → LAST_TASK anaphora
         │   ├─ 1 match → _dispatch_single() → (reply, snap_id)
         │   │                                    → _undo_keyboard if snap_id
         │   ├─ >1 match → _disambiguation_keyboard + store_pending_edit
         │   └─ 0 matches → error reply (no undo)
         └─ create/none → split_message → per-unit detect_intent loop
              ├─ edit units → execute_edit (inline)
              └─ create units → classify → persist
```

### Undo flow:
```
[Отменить] callback → cb_edit_undo
  → parse_edit_undo_callback(data) → snapshot_id
  → load TaskEditSnapshot WHERE id=snapshot_id AND user_id=user_id
  → TTL check (5 min)
  → _apply_undo(task, snap) — restore old_value
  → delete snapshot
  → remove inline keyboard
```

## 7 · Паттерны и best practices

### Executor tuple returns (PR-I4)
Все executors и `_dispatch_single` возвращают `tuple[str, int | None]`.
Вызывающие места (`execute_edit`, `cb_edit_resolve`) распаковывают:
```python
reply, snap_id = await _dispatch_single(task_id, user_id, intent)
kb = _undo_keyboard(snap_id) if snap_id else None
```

### parse_*_callback outside create_router
Регрессионный тест `test_callback_handlers_no_unguarded_int_parts` сканирует source `create_router()` на `int(parts[N])`.
Все parse-функции (`parse_task_callback`, `parse_edit_resolve_callback`, `parse_edit_undo_callback`) живут вне `create_router()` — try/except ValueError внутри.

### _apply_undo field dispatch
Каждый field type восстанавливается вручную через if-chain (не через setattr).
Для `due_at` и `horizon_id`/`category_id` нужен type conversion (datetime.fromisoformat, int).

### Session isolation
Executor: snapshot сохраняется в отдельной session_scope (внутри `_save_snapshot`), не в той же что мутация.
Undo handler: load + restore + delete snapshot в одной session_scope.

## 8 · Lessons Learned (PR-I4)

1. **Executor return type**: меняя `str → tuple[str, int | None]`, обновить ВСЕ callsites — execute_edit, cb_edit_resolve, тесты.
2. **Existing test assertions**: `assert kb is None` → `assert kb is not None` после добавления undo keyboard.
3. **LAST_TASK test pollution**: test_execute_edit_no_query использует user_id=1, после I4 тестов LAST_TASK может быть dirty — нужен `LAST_TASK.pop(1, None)`.
4. **int(parts[N]) regression**: parse_edit_undo_callback ОБЯЗАТЕЛЬНО вне create_router(), иначе regression test упадёт.
