## 2 · Детали PR-I4 (последний PR)

### 2.1 TaskEditSnapshot модель
`app/db/models.py` (326 строк) — новый класс:
```python
class TaskEditSnapshot(SQLModel, table=True):
    __tablename__ = "task_edit_snapshots"
    id: int | None = Field(default=None, primary_key=True)
    task_id: int  # FK → tasks.id ON DELETE CASCADE, indexed
    user_id: int  # FK → users.id, indexed
    field: str    # max_length=32 (status, title, priority, due_at, ...)
    old_value: str | None
    new_value: str | None
    created_at: datetime  # default=utcnow, for lazy TTL
```

### 2.2 Alembic migration 0011
`alembic/versions/2026_05_11_1700-0011_task_edit_snapshots.py`
- Ревизия `0011`, down_revision `0010`
- Написана вручную (autogenerate с SQLite = боль, см. Lessons Learned §8)

### 2.3 Executor return type change
**ВСЕ 9 executor'ов** теперь возвращают `tuple[str, int | None]`:
```python
# Было:
async def _execute_complete(task_id, user_id) -> str
# Стало:
async def _execute_complete(task_id, user_id) -> tuple[str, int | None]
```
Каждый executor вызывает `_save_snapshot()` после мутации.
Error/no-change cases возвращают `(reply_text, None)`.
`_dispatch_single()` тоже возвращает `tuple[str, int | None]`.

### 2.4 Snapshot helpers (`edit_executor.py`, 492 строки)
```python
_UNDO_TTL_SECONDS = 300  # 5 minutes

async def _save_snapshot(task_id, user_id, field, old_value, new_value) -> int | None
def _undo_keyboard(snapshot_id: int) -> InlineKeyboardMarkup
```

### 2.5 Undo handler (`callbacks.py`, 740 строк)
Callback data: `edit:undo:<snapshot_id>`
1. `parse_edit_undo_callback(data)` — **вне** `create_router()` (регрессионный тест `test_callback_handlers_no_unguarded_int_parts` сканирует source create_router на `int(parts[N])` — все parse-функции ОБЯЗАНЫ быть вне неё)
2. Load TaskEditSnapshot WHERE id=snapshot_id AND user_id=user_id
3. Lazy TTL: `(utcnow_naive() - created_at).total_seconds() > 300` → «Время для отмены истекло»
4. `_apply_undo(task, snap)` — restore old_value по field:
   - `status` → task.status = old_value
   - `deleted_at` → task.deleted_at = None
   - `title` → task.title = old_value
   - `priority` → task.priority = old_value
   - `due_at` → datetime.fromisoformat(old_value)
   - `horizon_id` → int(old_value)
   - `category_id` → int(old_value)
5. Delete snapshot row, remove inline keyboard

## 4 · Архитектура (итог I1–I4)

### 4.1 Intent detection flow
```
Текст → detect_intent(full text)
         ├─ edit intent (10 шт) → execute_edit()
         │   ├─ list_done → _execute_list_completed_today (read-only, no undo)
         │   ├─ query empty → LAST_TASK anaphora (TTL 60s)
         │   ├─ 1 match → _dispatch_single() → (reply, snap_id)
         │   │                                    → _undo_keyboard if snap_id
         │   ├─ >1 match → _disambiguation_keyboard + store_pending_edit
         │   └─ 0 matches → error reply (no undo)
         └─ create/none → split_message → per-unit detect_intent loop
              ├─ edit units → execute_edit (inline)
              └─ create units → classify → persist
```

### 4.2 Undo flow
```
User clicks [Отменить] → edit:undo:<snapshot_id>
  → cb_edit_undo
  → parse_edit_undo_callback(data) → snapshot_id
  → load TaskEditSnapshot WHERE id AND user_id
  → TTL check (5 min from created_at)
  → _apply_undo(task, snap) — restore old_value
  → delete snapshot row
  → edit_text(reply, reply_markup=None) — remove button
```

## 6 · Паттерны и best practices

### 6.1 Session isolation
Каждый executor принимает `(task_id, user_id)`, создаёт `session_scope()`, ищет Task свежую. **НИКОГДА** не передавать объекты Task между сессиями.

### 6.2 Executor tuple returns (PR-I4)
```python
reply, snap_id = await _dispatch_single(task_id, user_id, intent)
kb = _undo_keyboard(snap_id) if snap_id else None
```

### 6.3 parse_*_callback outside create_router
Регрессионный тест `test_callback_handlers_no_unguarded_int_parts` **сканирует** source `create_router()` на `int(parts[N])`.
Все parse-функции ОБЯЗАНЫ жить **вне** `create_router()` с try/except ValueError.

### 6.4 LAST_TASK
`time.monotonic()` для TTL — immune to system clock changes. Pop-on-use.

### 6.5 PENDING_EDITS
Pop-on-use: `pop_pending_edit()` removes the entry. Если callback после TTL → None → fallback.

### 6.6 E2E test mock sequence
Pipeline call order:
1. `detect_intent(full text)` — intent detection
2. `detect_reorder(text)` — legacy reorder (if create/none)
3. `split_message(text)` — split into units
4. `detect_intent(unit)` × N — per-unit intent (PR-I3)
5. `classify_intent(unit)` × N — classify each create-unit
6. `courier_respond(items)` — format reply (if not template_only)

### 6.7 Alembic
- `autogenerate` не работает с SQLite (alter column → crash). Для простых CREATE TABLE пиши миграцию вручную.
- Sequence: 0001–0011. Следующая: `0012`.
- Для теста: `DATABASE_URL="sqlite:///./test_migration.db" uv run alembic upgrade head`

## 8 · Lessons Learned (сессии 1-3)

1. **Secret name**: env var `GITHUB` (не `GUTHUB` — была опечатка в v17).
2. **Executor return type change**: меняя `str → tuple[str, int | None]`, обновить ВСЕ callsites — execute_edit, cb_edit_resolve, тесты.
3. **`assert kb is None` → `assert kb is not None`**: после добавления undo keyboard тесты на execute_edit нужно обновить.
4. **LAST_TASK test pollution**: `test_execute_edit_no_query` использует user_id=1, после I4 тестов LAST_TASK может быть dirty — нужен `LAST_TASK.pop(1, None)`.
5. **int(parts[N]) regression**: parse functions ОБЯЗАТЕЛЬНО вне create_router().
6. **Multi-intent E2E**: каждый split unit нуждается в своём `_intent_response("create")` в mock sequence.
7. **dateparser**: `languages=["ru"]`, `settings={"PREFER_DATES_FROM": "future"}`.
8. **Alembic autogenerate + SQLite**: `ALTER COLUMN` не поддерживается. Писать вручную для CREATE TABLE.
