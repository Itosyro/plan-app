## 1. В двух словах

Telegram-бот, который превращает поток мыслей (голос или текст) в структурированный план: задачи, заметки, напоминания. Под капотом — несколько LLM в цепочке «черновик → проверка», русский NLP для дат, и хранение в Postgres.

Изначально — личный инструмент для одного пользователя. Архитектурно подготовлено к 100+ пользователям (multi-tenancy, rate limit, очередь задач, идемпотентность webhook).

## 2. Главные сценарии

### 2.1. Утренний поток мыслей
Пользователь шлёт голосовое: «утром пробежка 5 км, в 11 совещание, до конца недели сдать отчёт, обед через час напомни».

Бот:
1. Транскрибирует через Whisper.
2. Разбивает на 4 интента (Splitter).
3. Каждому ставит категорию / горизонт / приоритет (Classifier).
4. Парсит время в структурированные даты («через час», «до конца недели») детерминированным кодом.
5. Извлекает напоминания.
6. Critic перепроверяет всю структуру.
7. Сохраняет 4 задачи + 1 напоминание.
8. Отвечает в Telegram дружелюбной квитанцией («Понял, добавил 4 задачи на сегодня и напоминание про обед через 1 час»).

### 2.2. Заметки и идеи
«Запиши идею: можно сделать функцию X для бота» → AI решает «это не задача, это заметка», создаёт запись в `notes` с категорией «идея», без горизонта.

### 2.3. Напоминание из текста
«Напомни мне через 43 минуты про звонок» → создаётся напоминание (с задачей или без — AI решает по контексту).

### 2.4. Перестановка задач голосом (Phase 2+)
«Задачу с отчётом перенеси на пятницу» → AI находит задачу, обновляет дедлайн.

### 2.5. Утренний / вечерний дайджест
В настроенное время бот сам присылает:
- Утром: задачи на сегодня + просроченное вчерашнее + горящие дедлайны недели.
- Вечером: что закрыто, что осталось, краткая сводка.

### 2.6. Ручной ввод и редактирование (Phase 3+)
Команды бота `/today`, `/week`, `/notes`, плюс инлайн-кнопки «выполнено / перенести / удалить».

### 2.7. Mini-app в Telegram (Phase 5)
Веб-интерфейс внутри Telegram: канбан + календарь (как Google Calendar) + список с фильтрами, переключаемые вкладки. На карточке задачи — её содержимое; в деталях — оригинальный транскрипт голоса/текста.

## 3. Сущности и понятия

| Сущность | Назначение |
|---|---|
| **User** | Telegram-пользователь, его настройки и часовой пояс |
| **Category** | Категория задачи (работа / здоровье / питание / идея / финансы / …). У каждого юзера свои. AI создаёт автоматически если не нашёл подходящую |
| **Horizon** | Горизонт планирования: сегодня / завтра / неделя / месяц / год / когда-нибудь. AI может создать кастомный |
| **Task** | Действие с категорией, горизонтом, приоритетом, опциональным дедлайном |
| **Note** | Заметка / мысль / идея — категоризируется, **горизонта нет** |
| **Reminder** | Напоминание (с задачей или самостоятельное), в конкретное время |
| **InboxEntry** | Сырой вход (текст или голос) до разбора |
| **AiRun** | Один запуск LLM-пайплайна: входные/выходные данные, использованные модели, время, статус |
| **TelegramUpdate** | Лог входящих апдейтов от Telegram (для идемпотентности webhook) |

Подробная схема — в [ARCHITECTURE.md](ARCHITECTURE.md#5-схема-бд).

## 4. Стиль ответа бота («курьер»)

Бот отвечает в два кадра:
1. **Подтверждение** («Понял», «Ясно», «Хорошо, мой господин», «Как скажете», «Принял», «Ок, шеф»…) — короткий «курьерский» ответ.
2. **Резюме сделанного** («добавил 5 задач на сегодня», «3 дела на вторник», «напомнил про обед через час»).

**Источник подтверждения выбирается рандомно при каждом ответе:**
- с вероятностью ~50% — берётся из встроенных шаблонов (≥30 фраз, минимум 5 на каждый стиль; стили: нейтральный / официальный «мой господин» / дружеский / шутливый / лаконичный / уважительный «как скажете»);
- с вероятностью ~50% — генерируется отдельным дешёвым вызовом LLM (короткая фраза, токенов мало) для живости.

Юзер может в `/settings` зафиксировать «только шаблоны» / «только AI» / «микс» (дефолт — микс). Стоимость +1 короткий запрос на ответ — допустимо, юзер этого хочет.

Резюме сделанного — всегда детерминированно собирается из персистнутых записей (без LLM).

## 5. Решения

| Тема | Решение |
|---|---|
| Язык | Python 3.12 |
| LLM-провайдер | Groq, 3 ключа (round-robin + резерв) |
| Модели Groq | Splitter — `llama-3.1-8b-instant`; Classifier — `llama-3.3-70b-versatile`; Critic — `qwen-qwq-32b` (reasoning); запасные — `llama-4-scout-17b-16e-instruct`, `gemma2-9b-it` (см. `ARCHITECTURE.md` §2.1) |
| Транскрибация | `whisper-large-v3` (точная, не turbo) |
| Деплой | Render Free + Neon Postgres (бесплатно, не умирает) |
| Доставка апдейтов | Webhook (не polling) |
| База | PostgreSQL через Neon |
| Язык интерфейса | Только русский (пока) |
| PR-стратегия | Маленькие PR пофазно |
| Стиль кода | docstring на английском, комментарии на русском в сложных местах |

## 6. Что в проекте *не* делаем

- Не делаем веб-приложение «как сайт» — только Telegram-бот + Telegram mini-app.
- Не используем Redis (на Render Free его нет; очередь — на Postgres).
- Не делаем мульти-LLM провайдеров на старте — только Groq. Локальные LLM — отдельный эксперимент в будущем.
- Не делаем оплату / авторизацию по email — авторизация только через Telegram.
- Не используем `smartkeyrouter` из старого репо — заменяем на простой роутер ключей в самом приложении.

## 7. Решения по уточняющим вопросам (snapshot ответов юзера)

| # | Вопрос | Решение |
|---|---|---|
| 1 | «На этой неделе» — дедлайн или метка? | **Комбо.** По умолчанию = дедлайн воскресенье 23:59. Можно для конкретной задачи переключить на «метка без срока». В `/settings` юзер выбирает дефолт. |
| 2 | «Через 5 минут пойти бегать» — задача + напоминание, только напоминание, или AI решает? | **AI решает по контексту** (вариант C). Если глагол явно действие — задача + напоминание; если «напомни …» — только напоминание. |
| 3 | Дефолтное смещение напоминания | **Если задача в течение текущего дня:** за 1 час и за 15 минут (двойное). **Если задача через несколько дней:** за 1 день и за 1 час. В `/settings` всё настраивается. |
| 4 | Critic — всегда / по уверенности / тумблер | **Тумблер в `/settings`** с дефолтом «по уверенности» (порог 0.7). |
| 5 | Утренний дайджест — 8:00 / 9:00? | **08:00** по умолчанию. В `/settings` меняется. |

При первом `/start` показываем onboarding-визард: имя → часовой пояс → дефолты (08:00 утренний, 21:00 вечерний дайджест, напоминания за 1ч+15мин, Critic «по уверенности», стиль курьера «микс»). В конце — «всё это меняется в `/settings`».
